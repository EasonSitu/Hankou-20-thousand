# 需求整理

## 产品定位

这是一个由个人发烧用户整理、面向普通大众使用的汉口近代建筑现场导览网页。

它应该像一个比普通攻略更懂建筑历史、比学术文章更好用、比粉丝站更克制的现场工具。

## 用户画像

- 普通游客：需要快速知道路线怎么走、哪些点必看、哪里可入内、哪里可跳过。
- 理性粉丝：看过“艺术与设计史”的“汉口两万步”，希望按视频路线复走。
- 建筑兴趣用户：想按银行、教堂、洋行、里份、工业建筑等类型横向比较。
- 现场使用者：需要手机上快速搜索、跳转和筛选。

## 当前范围

保留单文件 HTML 结构，重点增强现场可用性。

已实现：

- 地点搜索与搜索结果跳转。
- 搜索关键词高亮。
- 完整路线、精华路线、跟着视频走、按建筑类型筛选。
- 武汉警察博物馆、圣母无原罪堂等站点补充。
- 作者身份、资料时效、引用边界和现场秩序提示。

## 非目标范围

本项目不做：

- 账号登录。
- 打卡系统。
- 收藏和进度。
- 评论区。
- 问答游戏。
- 复杂地图导航。
- 过度饭圈化的称呼、徽章、应援感设计。

## 内容原则

- 每站至少回答：它是什么、为什么重要、现场看什么、是否适合入内。
- 视频路线是骨架，公开资料和书目用于校验背景与开放状态。
- 明确注明引用边界，不冒充官方导览或 UP 主原作。
- 开放/预约资料必须标注整理时间，提醒用户自行核验。

## 验收标准

- 搜索“巴公”“俄国”“银行”“教堂”“德租界”等能找到相关地点。
- 点击搜索结果能跳转到对应卡片。
- 切换路线模式不会破坏正文内容。
- 手机端无横向溢出，侧边导航和正文不互相遮挡。
- 页面可作为 GitHub Pages 静态站直接打开。
