# 汉口两万步网页：序言与后续优化说明

## 这次页面调整的方向

页面前半段不再放内部需求整理，也不再把读者拆成几类解释。首屏位置留给读者真正需要的信息：这是谁的路线、这个网页怎么用、哪里看地图、怎样选择路线。

致谢写法遵循四点：

- 原路线、实地拍摄、建筑辨认、路线组织和问题意识，明确归功于 B 站 UP 主“艺术与设计史”的“汉口两万步”系列。
- UP 主名称在网页中链接到个人主页，方便读者回到原作者内容。
- 网页署名只轻量出现为“B站用户 Eason-Situ 制作”，避免把个人贡献放在原作者之前。
- 明确说明这是非官方整理，不代表原 UP 主或官方导览。

## 页面已经移走的内容

这些内容不适合放在网页首屏，但适合留在维护文档里：

- 读者分类：普通游客、复走路线的观众、建筑兴趣用户、现场使用者。
- 内容目标：每站回答“它是什么、为什么重要、现场看哪里、是否适合入内”。
- 体验目标：搜索、跳转、筛选、手机阅读。
- 引用边界：不冒充原作，不代表官方导览，不替代现场公告。
- 现场原则：不打扰住户、办公机构、宗教活动和场馆秩序。
- 后续可深挖关键词。

## 路线模式的产品逻辑

当前网页的路线模式应让读者一眼明白：按钮会直接改变下方显示的站点，不只是文字推荐。

- 跟视频分集：默认模式。可按《汉口两万步》第 1 至 4 集筛选，适合粉丝复走。
- 半日精华：保留水塔街、江汉路和江汉关一线，适合时间或体力有限时使用。
- 一日精华：保留一日线关键站点，同时显示“一日节奏”。
- 完整路线：保留全部站点与可选点，适合出发前阅读或慢慢走。

高德地图二维码的作用单独说明：用高德扫码导入完整点位；网页内按钮用于筛选阅读内容。

## 后续优化方向

1. 每个站点补一条“30秒看懂”，优先写给现场读者，不写成论文摘要。
2. 把资料来源拆成单独维护文件，记录每条资料对应哪个站点、何时核验。
3. 给视频分集补时间点索引，只做指路，不搬运大段逐字稿。
4. 给半日、一日、完整路线分别补“适合谁”和“删点建议”。
5. 加入少量图片时，优先选能帮助识别建筑的图，不放纯装饰图。
6. 所有图片保留来源说明；若使用原 UP 主画面或截图，需要先确认授权边界。
7. 开放时间、预约方式、闭馆日每隔一段时间复核一次，避免旧信息误导读者。
8. 后续如果要贡献给原作者，可以先发一个克制版本：说明目的、来源、页面链接、可删除或修改的部分，不预设对方一定会接受。

网页是帮助读者理解和行走的现场手册，不替代原视频，也不替代官方资料。后续扩写时，优先补现场观察、路线判断和资料核验；少写宣传语，少写内部说明，少写只有制作者自己才关心的过程。
