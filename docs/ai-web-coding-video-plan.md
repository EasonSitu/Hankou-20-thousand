# 用 AI 做汉口口袋手册：视频脚本草案

## 核心切入点

这不是“AI 一键生成一个网页”的炫技视频，而是一个普通游客把一次真实旅行、一个专业 UP 主的路线和公开资料，整理成可在手机现场使用的小工具的过程。重点放在“AI 怎么帮人把热爱变成可分享的东西”：能查、能跳转、能筛路线、国内能访问，也尽量尊重原作者和现场秩序。

## 标题备选

1. 我用 AI 把一次武汉扫街，做成了手机能用的汉口建筑手册
2. 不是一键生成：我怎么用 AI 反复打磨一个旅行网页
3. 从 B 站路线到网页手册：用 AI 做一份能真正上街用的汉口导览
4. AI Web Coding 实战：把“汉口两万步”整理成一个口袋网页
5. 给老汉口建筑爱好者的小工具：一个 AI 辅助做出来的网页

## 短视频结构：60-90 秒

### 0-5 秒：问题钩子

画面：手机打开网页，快速展示搜索、路线选择、高德二维码。

旁白：

“我这次在武汉扫江汉路和沿江大道，发现一个问题：资料很多，但人在现场最需要的是一个能快速查、能跳转、能按路线走的手机手册。”

字幕：

“把旅行资料做成现场能用的网页”

### 5-18 秒：为什么做

画面：江汉路旧建筑照片、B 站视频页面、网页序言与致谢。

旁白：

“路线主要参考 B 站 UP 主‘艺术与设计史’的《汉口两万步》系列。原作者做了大量实地拍摄和建筑辨认，我做的不是替代视频，而是把路线顺序、公开资料和现场使用需求整理成一个非官方网页。”

字幕：

“尊重来源：路线与问题意识来自原作者”

### 18-38 秒：AI 参与了什么

画面：与 AI 对话片段、代码变化、页面从粗糙到清爽的对比。

旁白：

“AI 主要帮我做三件事：第一，把逐字稿和截图里的点位整理成结构化内容；第二，写成单文件 HTML，方便本地打开和国内托管；第三，根据现场体验一轮轮改文案和交互，比如去掉 AI 味、加搜索、加折叠按钮、把路线拆成视频分集、半日和一日。”

字幕：

“资料整理 / 网页编码 / 反复校对”

### 38-58 秒：网页怎么用

画面：搜索“银行”“俄国”“教堂”；点击分集；切换半日/一日；扫高德二维码。

旁白：

“现在它可以直接搜索建筑和关键词，也能按视频分集复走。时间不够，就切半日精华；想走完整一点，就看一日精华。高德二维码可以导入完整点位，网页里则负责解释每一站是什么、有什么背景、哪些信息需要现场核对。”

字幕：

“搜索 + 路线筛选 + 高德点位”

### 58-78 秒：态度和边界

画面：资料来源、GitHub、邮箱反馈。

旁白：

“我尽量把个人行程和主观指挥口吻删掉，只保留对普通游客有用的信息。这个网页开源，也欢迎继续改。它不是官方导览，也不代表原 UP 主，只是一个发烧用户做的小工具。”

字幕：

“非官方整理，欢迎共建”

### 78-90 秒：收尾引流

画面：网页地址、GitHub、评论区提示。

旁白：

“如果你也喜欢老汉口建筑，或者想看我怎么用 AI 一步步打磨这个网页，可以点进链接试用。也欢迎留言告诉我，还有哪些武汉路线值得做成这种口袋手册。”

字幕：

“链接见简介 / 欢迎反馈”

## 中长视频结构：3-5 分钟

1. 开场：旅行现场的真实痛点
   - 有路线、有视频、有攻略，但手机现场阅读不顺手。
   - 目标不是做百科，而是做“现场能用的读本”。

2. 材料来源
   - B 站 UP 主“艺术与设计史”的“汉口两万步”四期路线。
   - 逐字稿、截图、高德点位、公开资料、开放状态信息。
   - 说明尊重来源：原作者负责实地拍摄、建筑辨认和路线组织，本页只是二次整理。

3. AI Web Coding 流程
   - 先让 AI 从资料里提取点位、顺序、类型、关键词。
   - 再生成单文件 HTML，保证本地可打开、部署简单。
   - 通过浏览器截图不断指出问题：太粉丝向、AI 味重、爹味建议太多、搜索栏占空间、路线解释不清。
   - 每轮让 AI 改代码、跑测试、再看页面。

4. 最终页面能力
   - 搜索地点和关键词。
   - 跟视频分集、半日精华、一日精华。
   - 高德二维码导入完整点位。
   - 每站资料卡：建筑身份、历史背景、可观察要素、开放提示、资料来源。
   - 国内可访问，GitHub 开源。

5. 这个案例说明了什么
   - AI 不是替代判断，而是放大整理、编码和迭代能力。
   - 用户的审美和边界感很重要：哪里要删，哪里要克制，哪里要客观。
   - 好的 AI 协作不是一次生成，而是不断把“更像人、更有用、更尊重来源”的标准喂回去。

## 同类网站指导思想

1. 首屏先减法：标题、核心用途、必要入口和一条时效提示就够，不把需求整理、作者心路和长免责声明堆在最前面。
2. 情感要有，但不要抢主角：可以写对城市、街区和建筑的喜欢，不把页面写成个人游记。
3. 面向普通读者：默认读者没有看过完整背景，也不是作者本人；路线、搜索、跳转和资料卡要一打开就能用。
4. 尊重来源和劳动：原作者、资料出处、照片和路线来源要写清楚；自己的贡献轻量署名，不喧宾夺主。
5. 文案讲事实，不教训人：建筑详情写“它是什么、有什么历史、能看到什么、开放状态如何”，少用“你应该怎么看”“建议你如何体验”。
6. 路线要留余量：半日、一日、完整/分集路线要区分清楚，时间安排不要把人逼成打卡机器。
7. 手机优先：搜索、目录、路线按钮可以折叠；图片不能大裁切；文字和按钮不挤、不遮挡。
8. 国内访问友好：静态单页、轻资源、可本地打开，也能托管到国内访问更稳的平台。
9. 图片服务内容：图是证据和现场感，不是装饰。图文项目尤其要避免大图压住信息、裁掉关键建筑细节。
10. 保持可维护：来源、更新时间、反馈入口、GitHub 链接和邮箱要清楚，方便后续继续校对。
